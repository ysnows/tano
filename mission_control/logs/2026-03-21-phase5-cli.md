# 2026-03-21 — Phase 5: CLI Tooling

## Implemented commands

| Command | Status | Description |
|---------|--------|-------------|
| `tano create <name>` | Done | Scaffolds project from template |
| `tano doctor` | Done | Checks Bun, Xcode, simulators, Android SDK, Git |
| `tano dev` | Done | Starts Bun server, waits for ready, opens iOS Simulator |
| `tano build ios` | Done | Bun.build() bundles server + web, runs xcodebuild if ios/ exists |
| `tano run ios` | Done | Builds + installs + launches on simulator |
| `tano build android` | Stub | After Phase 6 |
| `tano run android` | Stub | After Phase 6 |
| `tano plugin` | Stub | Add/create plugin |

## Tested flow
```
tano create my-app → creates project with tano.config.ts, server.ts, web/index.html
cd my-app
tano build ios → bundles server (568B) + web assets to dist/
tano doctor → all 8 checks pass
```

## Key design decisions
- Zero dependencies — CLI uses only Bun built-ins
- Bun.build() for server bundling (minified)
- Web assets copied as-is (framework bundling deferred to framework's own tools)
- iOS Simulator management via xcrun simctl
- Falls back to dev mode when no Xcode project exists
